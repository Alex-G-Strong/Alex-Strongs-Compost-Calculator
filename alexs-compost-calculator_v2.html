📄 Full User Guide (README)

A single-page, offline-capable web app for designing multi-ingredient compost recipes. You tell it what feedstocks you have on hand and how much of each, rank what matters most (moisture, density, or C:N ratio), and it builds a phased "Operations Sheet" telling you exactly how much of each ingredient to load onto the pad. No install, no server, no internet connection required.

Quick Start

  1. Click + Start a New Recipe.
  2. Walk through the tabs in order: Constraints → Ingredients → Operations Sheet.
  3. Save the result to your Recipe Book so you can reload, print, or annotate it later.

Everything is saved locally in your browser — nothing is uploaded anywhere. Opens in dark mode by default; click the 🌙/☀️ toggle in the top right to switch.

1. Recipe Book

The home tab. Lists every recipe you've saved, showing the date, total volume, and final pile health metrics. From here you can:

  • Load Input — pull a saved recipe back onto the workbench to tweak and re-solve it.
  • + Note — attach a dated field note (and optionally a photo) to a recipe.
  • Export / Export All — download one recipe or your whole recipe book as a .json file.
  • Import — load a .json file exported from this app (yours or someone else's).
  • 🔗 Share — generates a link and QR code that encode just that recipe's ingredients and settings. Anyone who opens the link or scans the code gets it loaded straight into their own calculator.
  • ✕ — delete a recipe.

A status line under the header shows whether an automatic backup file is linked (see Automatic File Backup below).

2. Constraints — Site & Batch Settings

  • Measurement Unit — Cubic Yards, Cubic Meters, a custom loader bucket, Wheelbarrow, or 5-Gallon Bucket. Selecting Cubic Meters switches the whole batch-sizing workflow to metric (dimensions, target volume, and results all switch to m³/meters).
  • Define Batch Size By — Direct Volume, or Pile Dimensions (Length × Width × Height, using a parabolic pile profile). A warning appears if your height risks over-compaction.
  • Optimization Priorities & Ranges — drag Moisture, Density, and C:N Ratio into the order that matters most. The solver strictly prioritizes whichever card is on top — if your feedstocks can't satisfy all three at once, it sacrifices the lowest-ranked one first.

3. Ingredients

Build your list of available feedstocks: Name (autocompletes from the built-in library and your own saved items), C:N, Density, Moisture, Structural Index (1–10, resistance to compaction), and Available Inventory (the solver will never exceed this). For a feedstock not in the library, fill in the values yourself and click Save to Lib.

📚 Ingredient Library (button at the top of this tab) opens a larger reference tool: your own saved items, plus a 134-item Reference Database of real, named feedstocks grouped into collapsible categories, with a search bar and a Load button per item to drop it straight into a new row.

4. Operations Sheet (Results)

  • Status banner — green if the full target volume was built within your health ranges; red if it halted early.
  • Total Pile Yield and Overall Pile Health (moisture, density, C:N, Structure Score).
  • Phase Construction Breakdown — the pile is built in sequential phases as feedstocks run out; build it on the pad phase by phase, in order.
  • Print Operations Sheet — a clean, field-ready printout.
  • Save New Recipe / Update / Save as Copy to your Recipe Book.

How the Solver Works

  • Canonical metric backend — internally, all math runs in cubic meters, kg/m³, and kilograms, regardless of the unit you've selected. Every other unit converts at the display/input boundary, so solving the same physical batch in imperial or metric gives the same result.
  • Strict-to-relaxed priority weighting — your drag-and-drop order assigns penalty weights to deviations from each target, so the top-ranked metric is satisfied first.
  • Stochastic hill-climbing — the app randomly nudges the ingredient ratio and keeps any change that improves the fit, rather than a heavy linear solver.
  • Multi-phase fractional optimization — the pile is built in phases to respect hard inventory caps, mirroring how you'd actually build a pile on the pad as feedstocks run out.

Data & Privacy

Everything lives entirely in your browser, tied to that specific browser and device — nothing is sent to a server. This data will not appear if you open the calculator in a different browser, device, private/incognito window, or after clearing that browser's site data. Use Export All periodically to back up your recipes and move them between browsers/devices via Import.

Automatic File Backup (Chrome, Edge, Opera)

On Chromium-based browsers, the first time you save a recipe you'll be prompted to pick (or create) a .json backup file on disk. From then on, every save, update, delete, note, or import silently rewrites that file with your full, current recipe book — no manual Export needed. The Recipe Book tab shows the linked filename and lets you unlink or reconnect it any time.

Firefox has explicitly declined to implement the underlying API, and Safari has never supported it. On those browsers this feature isn't available, and a one-time notice explains that automatic backup needs a Chromium-based browser — Export/Import still works everywhere. Because of this, the calculator is designed to work best in Chrome, Edge, or Opera.

📲 How to share recipes between devices (Offline Sync)

Because this app stores data directly on your device, recipes do not automatically sync between your computer and phone. To transfer them:
1. On your computer, open the Recipe Book and click Export All (or export a single card). This downloads a small .json file.
2. Send that file to your phone. (Use AirDrop for Mac to iPhone, or Quick Share / Nearby Share for Windows to Android).
3. Open the Optimizer on your phone, click Import Recipe, and select the file you just transferred. It will instantly load into your Recipe Book.

🔎 Troubleshooting: Why can't I see my prior recipes or ingredients?

Q: I saved recipes/ingredients before, but they're gone now — what happened?
A: This app saves everything directly in your browser (recipes in IndexedDB, custom ingredients in localStorage) — nothing is uploaded to a server. That storage is tied to this specific browser, on this specific device. Your data will appear to be "missing" if you:

  • Open the calculator in a different browser (e.g. saved in Chrome, now opening in Edge/Firefox/Safari)
  • Open it on a different device or under a different Windows/Mac user account
  • Use a private/incognito window (this storage is wiped when the window closes)
  • Recently cleared your browser's site data / browsing history

Always open the calculator from the same browser to keep your data. To move data between browsers or devices, use Export All in the Recipe Book and Import it on the other side — see the sync FAQ above.

🏗️ What is the Structural Index?

The Structural Index (1-10) is a proxy for how well an ingredient resists compaction. Heavy, wet ingredients (food waste = 1) will compact and go anaerobic if the pile is built too high. Rigid carbon bulking agents (wood chips = 10) hold the pile open. The calculator ensures your overall pile maintains an average Structural Score above 5.0 to maintain Free Air Space (FAS).

🧠 ReadMe: Architecture & Simulator Math

The "Strict-to-Relaxed" Priority System
In field composting, perfect chemistry is often impossible given available feedstocks. By allowing the user to visually rank priorities via drag-and-drop, the mathematical fitness function penalizes deviations from the top priority heavily, forcing the algorithm to satisfy the most critical parameter first.

Weighted Hill-Climbing Algorithm
Standard matrix solvers are complex to implement without heavy external libraries. This tool uses a stochastic hill-climbing search, which is lightweight, fast, and easily handles custom weighted penalties for non-linear metrics like ratio-based constraints.

Multi-Stage Fractional Optimization (Inventory Caps)
The calculator builds the pile in "Phases." It optimizes a ratio, scales it until an ingredient runs out, zeroes that ingredient, and recalculates a new ratio to fulfill the remaining volume. This mirrors how an operator actually builds a pile on the pad, ensuring the total pile meets health standards even if feedstocks run out mid-build.
